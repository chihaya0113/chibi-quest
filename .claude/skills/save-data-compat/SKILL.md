# save-data-compat — セーブデータと進捗の保護

大原則: **子供が積み上げた進捗(XP、レベル、解放済み単元、獲得済み選手カード)を、アップデートで減らさない・消さない。** セーブデータが壊れる=子供の数週間の努力が消える事故、として扱う。

## セーブデータ構造を変更するとき(src/storage.js)

1. 既存パターンに従う: `CURRENT_MIGRATION_VERSION` を+1し、旧バージョンの state を新構造へ変換する移行処理を追加する。`defaultState` への項目追加だけでも、既存セーブにその項目が無いケースの初期化を必ず書く。
2. 移行の検証: 旧バージョン相当の state を localStorage に手で仕込んでから起動し、(a) エラーなく読み込めること (b) XP・レベル・解放単元・獲得カードが移行前と同じ値であることを確認する。
3. 壊れたJSON・未知バージョンを読んだときに `defaultState` で上書きする分岐がある場合、その分岐が「本当に復旧不能な場合」しか通らないことを確認する(安易な上書きは全進捗消失になる)。

## ゲームバランス・カードプールを変更するとき

- `players.js`(選手カードプール)から選手を削除・ID変更する場合: セーブ内の獲得済みカードIDが参照切れにならないか確認する。削除するなら移行処理で代替(別カード付与など)を与える。
- XP・レベル計算式の変更: 既存プレイヤーの現在XPで再計算し、**レベルが下がる子が出ないか**を代表値(低・中・高XP)で確認する。下がる場合はその変更をしないか、移行で補填する。
- 排出率・解放条件を厳しくする変更は、すでに獲得・解放済みのものには適用しない。

## 禁止事項

- 移行処理なしで `STORAGE_KEY` や state 構造を変える
- 旧セーブがある状態での起動確認をせずにリリースする(release-pwa 参照)
- 獲得済みの進捗・カード・解放単元を遡って取り上げる変更
