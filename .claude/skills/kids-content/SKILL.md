# kids-content — 問題・文言の追加と修正

対象ユーザー: 小学3年生(8〜9歳)。保護者向けの画面・文言は大人向けの日本語でよい(子供向けと区別する)。

問題の設計(難易度・類題防止・面白さ・メタデータ)は chibique-question-design-core Skill が正本。問題の新規作成・刷新ではそちらも必ず併用する。このSkillは表記・生成手順・auditを担当する。

## 表記ルール(子供が読む文言すべて)

- 漢字は小学3年生までの配当漢字を目安にし、それ以上はひらがなで書く(迷ったらひらがな)
- 1文は短く(目安30文字以内)。二重否定や受け身の多用をしない
- まちがえたときの文言は責めずに励ます(「おしい!つぎはできるよ」系。「ダメ」「まちがい」で終わらせない)
- 学習範囲外のカタカナ語・英語を説明なしで使わない

## 問題データの変更手順

1. 問題ファイル(`src/data/questions/grade3` 配下)を直接編集せず、生成スクリプト(`scripts/generate-*.mjs`)を修正して `npm run generate:questions` で再生成する(スクリプトが正)。
2. 生成後、必ず `npm run audit:questions` を実行し、全チェック(問題数、必須フィールド、4択の重複、正解の有無、説明文、問題文パターンの重複)が通ることを確認する。
3. 4択の品質基準:
   - 誤答は「ありがちな間違い」にする(計算ミス、単位の取り違え、似た読みの語など。明らかに変な選択肢で埋めない)
   - 正解が形式的に浮かないようにする(1つだけ極端に長い・短い選択肢にしない)
4. 教科ごとの問題数(各260問)や出題形式を変える場合、出題側の前提(1回10問)と audit の期待値に影響しないか確認する。
5. 動作確認: `npm run serve` で起動し、変更した単元を実際に10問プレイして、表示崩れがなく解説が正しいことを確認する。

## 学習内容の正確性

- 各教科の内容は小学3年生の学習範囲に合わせる。範囲外の内容を入れる場合は、報告に「小3範囲外」と明記してユーザーの確認を取る。
- 解説文は「なぜその答えになるか」を子供が読める言葉で書く(答えの繰り返しだけにしない)。

## 禁止事項

- `audit:questions` を通さずに問題データをcommitする
- 生成スクリプトを経由せず、生成された問題ファイルを直接書き換える
- 子供向け文言に小学4年生以上の漢字をふりがななしで使う
